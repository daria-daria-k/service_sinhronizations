# Сервис синхронизации файлов с Яндекс.Диском

Консольное приложение на Python, которое в фоновом режиме отслеживает изменения файлов в указанной локальной папке и автоматически синхронизирует их с папкой на Яндекс.Диске.

## Возможности

- При появлении нового локального файла — загрузка в облако
- При изменении локального файла — перезапись копии в облаке
- При удалении локального файла — удаление копии в облаке
- На старте — первичная синхронизация (содержимое удалённой папки приводится в соответствие локальной)
- Логирование событий с уровнями `INFO` и `ERROR` в файл и в `stdout`
- Ошибки сети и доступа к файлам не прерывают работу программы
- Архитектура с абстрактным классом `CloudStorage` — облачное хранилище легко заменить (Google Drive, Dropbox и т. д.)
- Автоматическая ротация лог-файла (максимум 5 МБ, хранится 3 файла)

## Стек

- Python 3.10+
- [`requests`](https://pypi.org/project/requests/) — единственная сторонняя зависимость
- Стандартная библиотека: `configparser`, `os`, `logging`, `unittest`

## Получение OAuth-токена Яндекс.Диска

1. Откройте [oauth.yandex.ru](https://oauth.yandex.ru/) и нажмите **«Создать приложение»**.
2. Заполните форму:
   - **Название**: любое (например, `File Sync Service`).
   - **Платформы**: отметьте **«Веб-сервисы»** и в поле **Redirect URI** укажите `https://oauth.yandex.ru/verification_code`.
   - **Доступ к данным**: в разделе **«Яндекс.Диск REST API»** включите права **«Доступ к информации о Диске»** и **«Доступ к Яндекс.Диску для приложений»** (или **«Чтение всего Яндекс.Диска»** + **«Запись в любом месте на Яндекс.Диске»**).
3. Сохраните приложение — на странице приложения появится **ClientID**.
4. Перейдите по ссылке (вместо `<ClientID>` подставьте свой):

   ```
   https://oauth.yandex.ru/authorize?response_type=token&client_id=<ClientID>
   ```

5. Подтвердите доступ — Яндекс отобразит ваш OAuth-токен. Скопируйте его в `config.ini`.

> Токен действует около года; после истечения повторите шаг 4.

## Конфигурация

Скопируйте шаблон и заполните своими значениями:

```bash
cp config.ini.example config.ini
```

Параметры в секции `[sync]`:

| Параметр      | Описание                                                              |
|---------------|-----------------------------------------------------------------------|
| `local_path`  | Абсолютный путь к синхронизируемой папке на компьютере                |
| `remote_name` | Имя/путь папки на Яндекс.Диске (создастся автоматически)              |
| `token`       | Ваш OAuth-токен Яндекс.Диска                                          |
| `period`      | Период проверки изменений в секундах                                  |
| `log_path`    | Путь к файлу лога                                                     |

> Файл `config.ini` находится в `.gitignore` — реальный токен не попадёт в репозиторий. Перед сдачей дополнительно убедитесь, что в коммиты не утёк токен.

## Запуск

```bash
python main.py
```

Программа стартует в бесконечном цикле, отчёт о синхронизациях идёт в файл `sync.log` и в консоль. Остановить — `Ctrl+C`.

Пример строк лога:

```
2026-04-26 19:30:12 [INFO] Программа запущена. Синхронизируемая папка: /Users/example/SyncFolder
2026-04-26 19:30:13 [INFO] Загружен новый файл: notes.md
2026-04-26 19:30:43 [INFO] Обновлён файл: notes.md
2026-04-26 19:31:13 [INFO] Удалён файл: old.txt
2026-04-26 19:31:43 [ERROR] Не удалось обновить файл report.pdf: HTTP 503 — ...
```

## Тесты

```bash
python -m unittest discover -s tests
```

Тесты используют `unittest.mock` и не требуют реального токена/сетевого доступа.
